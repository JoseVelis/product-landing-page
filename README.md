# Inchi Fit - Landing Page

Este proyecto es una landing page para **Inchi Fit**, una marca de mantequilla de maní 100% natural. La página está diseñada para ser accesible, semántica y responsive, cumpliendo con los requisitos de la tarea.

---

## Estructura Semántica

La estructura semántica del HTML fue diseñada para cumplir con los estándares de **HTML5** y garantizar la **accesibilidad**. A continuación, se justifican las etiquetas utilizadas:

### **1. `<header>`**
- **Uso**: Contiene el menú de navegación principal.
- **Justificación**: El `<header>` es la etiqueta semántica adecuada para agrupar el contenido introductorio o de navegación. Aquí se incluye el logo y los enlaces de navegación.

### **2. `<nav>`**
- **Uso**: Define el menú de navegación.
- **Justificación**: La etiqueta `<nav>` se utiliza para agrupar enlaces de navegación, lo que mejora la accesibilidad y la estructura semántica.

### **3. `<main>`**
- **Uso**: Contiene el contenido principal de la página.
- **Justificación**: El `<main>` es la etiqueta semántica que engloba el contenido principal de la página, como la sección "Hero" y las secciones informativas.

### **4. `<section>`**
- **Uso**: Divide el contenido en secciones temáticas (Hero, Características, Envíos, Pagos).
- **Justificación**: Cada `<section>` representa una parte independiente del contenido, lo que mejora la organización y la accesibilidad.

### **5. `<footer>`**
- **Uso**: Contiene información de contacto y enlaces importantes.
- **Justificación**: El `<footer>` es la etiqueta semántica adecuada para el pie de página, donde se incluye información secundaria.

### **6. `<h1>`, `<h2>`, `<h3>`**
- **Uso**: Jerarquía de encabezados.
- **Justificación**: Los encabezados se utilizan para estructurar el contenido y mejorar la accesibilidad para los lectores de pantalla.

### **7. `<ul>` y `<li>`**
- **Uso**: Listas de características, opciones de envío y métodos de pago.
- **Justificación**: Las listas son la forma semántica correcta de presentar elementos relacionados.

### **8. `<img>` con `alt`**
- **Uso**: Imágenes descriptivas.
- **Justificación**: El atributo `alt` mejora la accesibilidad al proporcionar una descripción textual de las imágenes.

### **9. `<button>`**
- **Uso**: Botón de alternancia para el modo oscuro.
- **Justificación**: El `<button>` es la etiqueta semántica adecuada para elementos interactivos.

## Prompt Usado con la IA

Para obtener sugerencias sobre la estructura semántica y la accesibilidad, se utilizó el siguiente prompt con la IA:


### **Respuesta de la IA**
La IA sugirió:
1. Usar etiquetas semánticas como `<header>`, `<main>`, `<section>`, y `<footer>`.
2. Asegurar que todas las imágenes tengan el atributo `alt`.
3. Utilizar una jerarquía coherente de encabezados (`<h1>`, `<h2>`, `<h3>`).
4. Evitar el uso excesivo de `<div>` cuando haya etiquetas semánticas más apropiadas.

### **Validación de la Respuesta**
- **Verificación manual**: Se revisó que las etiquetas sugeridas por la IA fueran apropiadas para cada sección del contenido.
- **Herramientas de validación**: Se utilizó la herramienta [WAVE](https://wave.webaim.org/) para verificar la accesibilidad y asegurar que no hubiera errores.
- **Pruebas en navegadores**: Se probó la página en diferentes navegadores (Chrome, Firefox, Edge) para garantizar la compatibilidad.

## Cómo Validé la Respuesta de la IA

1. **Revisión Manual**:
   - Verifiqué que las etiquetas semánticas sugeridas por la IA fueran coherentes con el contenido de la página.
   - Aseguré que los atributos `alt` de las imágenes fueran descriptivos.

2. **Herramientas de Validación**:
   - Usé [WAVE](https://wave.webaim.org/) para verificar la accesibilidad y detectar posibles problemas.
   - Utilicé el validador de HTML del W3C para asegurar que el código estuviera libre de errores.

3. **Pruebas en Navegadores**:
   - Probé la página en Chrome, Firefox y Edge para garantizar que la estructura y los estilos se vieran correctamente en todos los navegadores.

4. **Lectores de Pantalla**:
   - Usé el lector de pantalla NVDA para verificar que la página fuera accesible para usuarios con discapacidades visuales.
